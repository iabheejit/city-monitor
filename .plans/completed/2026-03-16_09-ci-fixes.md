# Plan 09 — CI-Blocking Fixes (Lint + Test)

Type: bugfix
Complexity: simple
Files affected: 2

## Problem

Two CI-blocking issues:

1. **Lint error (C1):** In `packages/web/src/lib/api.ts`, line 12 has an `import type` that pulls in `NewsItem` and `NewsDigest` from `@city-monitor/shared`. These same types are already re-exported on line 14 via `export type { NewsItem, NewsDigest } from '@city-monitor/shared'`. The `@typescript-eslint/no-unused-vars` rule flags `NewsItem` as unused (it is never referenced locally). `NewsDigest` IS used locally (lines 17, 159), but the remaining three types on line 12 (`WeatherData`, `ApiResponse`, `HistoryPoint`) are also used locally and must stay.

2. **Test failure (I1):** All 6 `App.test.tsx` tests crash because `Shell.tsx` uses `new ResizeObserver(...)` (line 21), which does not exist in jsdom.

## Changes

### File 1: `packages/web/src/lib/api.ts`

**Remove only `NewsItem` from line 12.** `NewsDigest` must remain because it is used locally in `BootstrapData` (line 17) and `getNewsDigest` (line 159) — `export type` re-exports do NOT introduce bindings into local scope.

Before:
```ts
import type { WeatherData, ApiResponse, HistoryPoint, NewsItem, NewsDigest } from '@city-monitor/shared';
```

After:
```ts
import type { WeatherData, ApiResponse, HistoryPoint, NewsDigest } from '@city-monitor/shared';
```

> **Note:** If the lint error actually flags `NewsDigest` too (because the linter considers the re-export on line 14 as "usage"), then also remove `NewsDigest` from line 12. If that causes a type error, move `NewsDigest` from line 14's re-export into line 11's re-export and add it back to line 12's import. Verify by running `npx turbo run lint --filter=@city-monitor/web` after the change.

### File 2: `packages/web/src/test-setup.ts`

Add a global `ResizeObserver` mock before the `matchMedia` mock (around line 91). The mock provides the three methods that `Shell.tsx` calls: `observe`, `unobserve`, `disconnect`.

```ts
// Mock ResizeObserver for jsdom environment (used by Shell.tsx)
global.ResizeObserver = vi.fn().mockImplementation(() => ({
  observe: vi.fn(),
  unobserve: vi.fn(),
  disconnect: vi.fn(),
})) as unknown as typeof ResizeObserver;
```

Place this before the `matchMedia` mock since both are global API polyfills for jsdom.

## Verification

1. `npx turbo run lint --filter=@city-monitor/web` — no errors in `api.ts`
2. `npx turbo run test --filter=@city-monitor/web` — all 6 `App.test.tsx` tests pass
3. `npx turbo run typecheck` — no type errors
